# 摘要

这不是一个如何编写内核代码的指南。它致力于理解 Linux 内核是如何思考的。

在系统编程中，人们很容易迷失在符号、头文件以及具体实现之中。但在代码之下，内核本质上是一个结构化且具备响应性的系统——由上下文所主导，以分离为基础，并以精确的设计意图处理从内存到调度的所有事务。

这个系列面向那些希望在接触源码之前，先构建内核工作方式的心智模型的人。无论你是第一次探索 Linux 内部机制，还是带着新的问题再次回到这里，这里关注点在于行为，而非语法。

每一篇文章最初都源于一次独立思考。它们组合起来，呈现出一张概念性的地图。
不是函数调用，而是内核如何响应、如何约束、如何隔离、如何提供服务。

内核运行着一切。让我们开始理解它是如何运行的。
